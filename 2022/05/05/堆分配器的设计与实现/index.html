堆分配器的设计与实现

堆分配器的设计与实现

前言

Linux环境高级编程

C语言

一、需求分析

1.1 总体要求

实现一个堆分配器，即实现内存分配和释放的功能

1.2 基本要求

实现的内存分配函数名为MyMalloc，内存释放函数名为MyFree，这两个函数原型同malloc和free；

使用mmap函数从内核分配整个堆空间，而MyMalloc和MyFree管理该空间；

考虑内存碎片问题

要考虑并发性问题，即多个线程同时分配和释放内存的场景

二、总体设计

2.1 总体设计概述

2.2 具体模块设计

2.2.1 数据结构

堆的管理涉及到两个数据结构，堆头(heap_header)以及内存块头(block_header)。

堆头 heap_header

结构体中包括pre_free_block和heap_size两个数据成员。pre_heap_break为当前第一个空闲内存块（本设计内存块分配策略为循环适应法），heap_size为堆的大小（包括堆头）。

内存块头

结构体中包括block_size和block_flag两个数据成员。block_size为内存块的大小，block_flag为当前内存块是否空闲，0为空闲，1为已分配。

三、详细设计与实现

3.1 初始化堆 init_heap()

使用mmap()函数从内核分配整个堆空间，将返回的地址赋值给堆首地址。此时堆的大小为file_size – heap_header_size，第一块空闲内存块首地址为heap + heap_header_size，空闲块大小为file_size - heap_header_size - block_header_size，并将block_flag置为0。初始化堆的过程中同时将首个内存块地址记录，后续碎片整理以及其他模块中需要用到。

3.4 内存分配 MyMalloc()

代码如下：

何时分配内存块？当前的内存块头的block_flag == 0，block_size > mem_size(申请内存块大小)时，可将当前内存块分配；

如果可以分配，做些什么？将当前内存块的block_flag = 1，表示已被分配，获取下一块内存块头地址，为next_block = current_block + block_header_size + mem_size，计算当前与下一块内存块的大小，current_block->block_size = mem_size，next_block->block_size = current_block->block_size - block_header_size - mem_size，并将下一块内存块的block_flag = 0，表示为空闲内存块；

如果不能分配，做些什么？通过get_next_block()函数获取下一块内存块，继续重复上算法；

什么时候终止（即所申请内存大小不能满足）？一开始我们初始化了一个origin_block用来存放分配时第一个访问的内存块，即pre_free_block，如果一直未能获取满足要求的内存块，一直通过get_next_block()获取下一块内存块直到循环到了最开始的内存块，此时表明堆中已没有满足要求的内存块，此时退出函数，return NULL，表示申请失败。

3.6 并发性问题

调用pthread.h库，利用pthread_mutex_lock()以及pthread_mutex_unlock()两个函数完成信号量的操作，即PV操作。

本设计中内存块资源为临界资源，因此考虑并发性问题（多个线程同时分配和释放内存的场景）就是考虑关于内存块这个临界资源的操作。本设计中涉及到内存块的操作的仅有defrag()，MyMalloc()，MyFree()三个模块，而且defrag()模块是在MyMalloc()模块内调用，故只需要将这两个模块进行的内存块操作夹在P操作和V操作之间即可，如下:
